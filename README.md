# MSCI (Microservice Continuous Integration) 

Msci is created to be used in software projects based on microsevice architecture to help versioning and continuous integration.

# Problem  
  Sample Development Environment ::  
  25+ Software Developers,
  150+ Projects,
  25+  Deployable Projects
  
  Task of deciding which projects to be released and which projects to use updated releases.

# Features
  1. Build reverse dependency graph. List projects which depends on a specific projects.
  2. List projects which needs to be  released or deployed.

# Future Features
  1. Auto-release project.
